A. The event service is a mechanism for Redfish. Events will be sent out through https to the web
server that is subscribed to by the users.
First, user needs to add a subscription to inform Redfish who will receive this event.
After user adds subscriptions, following event will be send out.
+------------------------+-----------------+-------------------+--------------------+---------+
|    Events Types        | Resource Added  | Resource Removed  |  Resource Updated  | Alert   |

B. The demo test commands usage is according to the event type subscription/unsubscription:
1. subscribe the "ResourceAdded" event type
Test Commands:
./dm attach <device ip address>:<device PSME listen port>:<device query interval>
(ex: ./dm attach 172.17.8.40:8888:120)
./dm setsessionservice <device ip address>:<device PSME listen port>:<HTTP Token or "">:true:<session timeout>
(ex: ./dm setsessionservice 172.17.8.40:8888:"":true:300)
./dm logindevice <device ip address>:<device PSME listen port>:<user name>:<user password> | cut -d' ' -f4
(ex: token=`./dm logindevice 172.17.8.40:8888:admin:redfish) ==> admin/redfish is PSME default user name/password.
./dm sub <device ip address>:<device PSME listen port>:<HTTP Token>:<HTTP or HTTPS>:<Event Notification server ip address>:<port>:<event type>
(ex: ./dm sub 172.17.8.40:8888:53831af4cd69f722ebc6578c56ddefac:https:0.0.0.0:3180:add) ==> The 0.0.0.0 ip address means the "demotest" is a the notification server.

C. The subscription/unsubscription format of event types sends to PSME:
1. Add a subscription about ResourceAdded
 - URL :  /redfish/v1/EventService/Subscriptions
 - Required Privilege : ConfigureComponents
 - Method : POST
 - Payload :
   {
   "Name": "EventSubscription ResourceAdded TEST",
   "Destination": "https://172.17.10.60:8889",
   "EventTypes": [
   "ResourceAdded"
   ],
   "Context": "ResourceAdded notification message",
   "Protocol": "Redfish"
   }

 - Response : 201 CREATED

2. Add a subscription about ResourceRemoved
  - URL : /redfish/v1/EventService/Subscriptions
  - Required Privilege : ConfigureComponents
  - Method : POST
  - Payload :
    {
    "Name": "EventSubscription ResourceRemoved TEST",
    "Destination": "https://172.17.10.60:8889",
    "EventTypes": [
    "ResourceRemoved"
    ],
    "Context": "ResourceRemoved notification message",
    "Protocol": "Redfish"
    }

    - Response : 201 CREATED

3. Add a subscription about Alert
  - URL : /redfish/v1/EventService/Subscriptions
  - Required Privilege : ConfigureComponents
  - Method : POST
  - Payload :
    {
    "Name": "EventSubscription Alert TEST",
    "Destination": "https://172.17.10.60:8889",
    "EventTypes": [
    "Alert"
    ],
    "Context": "Alert notification message",
    "Protocol": "Redfish"
    }

  - Response : 201 CREATED

4. Add a subscription about ResourceUpdated
  - URL :  /redfish/v1/EventService/Subscriptions
  - Required Privilege : ConfigureComponents
  - Method : POST
  - Payload :
    {
    "Name": "EventSubscription ResourceUpdated TEST",
    "Destination": "https://172.17.10.60:8889",
    "EventTypes": [
    "ResourceUpdated"
    ],
    "Context": "ResourceUpdated notification message",
    "Protocol": "Redfish"
    }

  - Response : 201 CREATED

5. Alert event send out once every 30 seconds to listener until alert situation resolved. Get subscription events:
  - URL : /redfish/v1/EventService/Subscriptions
  - Required Privilege : ConfigureComponents
  - Method : POST
  - Payload : 
    {
    "@odata.context": "/redfish/v1/$metadata#EventService.EventService",
    "@odata.id": "/redfish/v1/EventService/Subscriptions",
    "@odata.type": "#EventDestinationCollection.EventDestinationCollection",
    "Name": "Event Subscriptions Collection",
    "Description": "Collection of Event Subscriptions",
    "Members@odata.count": 4,
    "Members": [
    {
    "@odata.id": "/redfish/v1/EventService/Subscriptions/1"
    },
    {
    "@odata.id": "/redfish/v1/EventService/Subscriptions/2"
    },
    {
    "@odata.id": "/redfish/v1/EventService/Subscriptions/3"
    },
    {
    "@odata.id": "/redfish/v1/EventService/Subscriptions/4"
    }
    ]}

  - Response : 200 OK

6. unsubscription events (ex: subscription id 1)
- URL : /redfish/v1/EventService/Subscriptions/1
- Required Privilege : ConfigureComponents
- Method : Delete
- Payload :
  {}

- Response : 204 NO_CONTENT

7. The deivce temperature could configure the level of alert temperature.
  - URL : /redfish/v1/Chassis/1/Thermal
  - Method : Patch 
  - Payload :
  {
    "Temperatures":
      {
        "MemberId": "1",
        "UpperThresholdNonCritical": 83,
        "LowerThresholdNonCritical": 33
      }
  }

  - Response : 200 OK

8. Set CPU loading percentage warning UpperThresholdNonCritical
  - URL     : /redfish/v1/Systems/1 
  - Required Privilege :  ConfigureManager 
  - Method  :  Patch 
  - Payload :  
  
  {
    "CpuStatus":
      {
        "UpperThresholdNonCritical": 83
      }
  }

  - Response : 200 OK

9. Set free memory percentage warning LowerThresholdNonCritical
  - URL     : /redfish/v1/Systems/1 
  - Required Privilege :  ConfigureManager 
  - Method  :  Patch 
  - Payload :  
  
  {
    "MemoryStatus":
      {
        "LowerThresholdNonCritical": 30 
      }
  }

  - Response : 200 OK

10. Set StorageStatus usage percentage warning UpperThresholdNonCritical
  - URL     : /redfish/v1/Systems/1 
  - Required Privilege :  ConfigureManager 
  - Method  :  Patch 
  - Payload :  
  
  {
    "StorageStatus":
      {
        "UpperThresholdNonCritical": 83
      }
  }

  - Response : 200 OK
